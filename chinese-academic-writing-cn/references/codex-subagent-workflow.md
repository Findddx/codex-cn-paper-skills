# Codex 子代理写作与审阅工作流

本文件用于把“多模型交叉润色”“固定外部 API 校对”等专用流程改造成通用 Codex 工作流。核心思路是：同一份论文由不同审阅角色分别处理不同风险，主代理统一裁决和整合。

## 何时使用

以下任务应启用 subagents 或分轮模拟：

- 学位论文、期刊论文、长篇综述、政策法学交叉论文等高风险文本。
- 用户要求“终稿”“投稿前”“降 AI 感”“全面润色”“引用核查”“逻辑重构”。
- 文本超过单一段落，且同时存在结构、证据、语言和格式要求。
- 修改会影响章节顺序、研究结论、引用编号或 Word 格式。

## 角色设计

### 结构审阅者

输入：题目、摘要、目录、章节文本或用户要求。

任务：

- 判断研究问题是否清晰，章节是否围绕同一问题展开。
- 检查引言、综述、方法、分析、结论之间是否有承接。
- 标出重复章节、跳跃段落和缺少过渡的位置。
- 给出不超过三条结构调整建议，避免重写全文。

输出：结构问题清单、建议调整顺序、需保留的原文内容。

### 证据映射者

输入：正文、引用标记、参考文献、数据表、案例材料或原文摘录。

任务：

- 逐条核查核心判断是否有证据。
- 标出引用堆叠、引用错位、证据不足和外推过度。
- 区分文献综述、事实依据、方法来源和观点支持。
- 对无法确认的引用提出“保留但需核对”或“删除/降格表达”的建议。

输出：观点-证据映射表、风险句、建议修订方式。

### 方法严谨性审阅者

输入：方法章节、模型、变量、公式、样本说明、数据来源、研究设计。

任务：

- 检查研究对象、样本范围、时间范围、变量定义和指标构造是否可复核。
- 检查公式符号、参数解释、单位、数据处理流程是否完整。
- 标出把描述性分析写成因果推断、把个案观察写成普遍结论的地方。

输出：方法风险清单、必须补充的信息、建议改写句。

### 学术表达润色者

输入：待润色正文和不可改动要求。

任务：

- 减少任务叙述、AI 口吻、空泛承接和模板句式。
- 保留原文核心观点、事实、引用标记和章节编号。
- 把口号式表述改成对象明确、机制清晰、限定充分的学术表达。
- 不追求华丽修辞，不制造未经证实的新概念。

输出：修订稿、关键改动说明、未处理风险。

### 终稿验收者

输入：拟交付终稿、引用与格式要求。

任务：

- 按最终验收清单检查论证、引用、术语、语体、格式口径。
- 专门检索“不是”“并非”“而是”及其组合，判断是否存在模板化滥用。
- 标出必须修改的问题和可选优化建议。

输出：是否可交付、必须修改项、建议修改项。

## 主代理整合规则

- 主代理不得直接拼接 subagent 文本。必须统一术语、引用、语气、章节承接和格式。
- 如果 subagent 意见冲突，以证据和用户要求优先；不能确认时保守处理并向用户说明。
- 子代理提出的新事实、新文献、新数据或新判断，必须经过来源核验后才能进入正文。
- 对长文可按章节循环执行，不必一次性把全文交给所有角色。

## 推荐顺序

1. 主代理整理任务、材料和不可改动项。
2. 结构审阅者先判断章节功能。
3. 证据映射者核查关键判断。
4. 方法严谨性审阅者处理研究设计和分析章节。
5. 学术表达润色者进行句段级改写。
6. 主代理整合终稿。
7. 终稿验收者做全局检查。
8. 主代理修复必须修改项并交付。

## 提示词模板

结构审阅：

```text
你是中文学术论文结构审阅者。只检查研究问题、章节功能、段落顺序和承接关系，不润色句子。请输出：1. 结构问题；2. 必须调整项；3. 可保留内容。
```

证据映射：

```text
你是中文论文证据映射审阅者。请逐条核对正文判断与引用、数据、案例或原文材料的关系。不要新增事实。输出观点-证据映射表，并标出引用错位、引用堆叠和证据不足。
```

表达润色：

```text
你是中文学术表达润色者。请在不改变事实、引用标记和核心观点的前提下，降低 AI 感和口号化表达。避免模板化转折，尤其不得滥用“不是/并非……而是……”。输出修订稿和关键改动说明。
```

终稿验收：

```text
你是中文论文终稿验收者。请按论证、证据、引用、术语、语体和格式口径审查全文。必须单列“不是/并非……而是……”审查结果：逐处判断是否确有概念辨析、批判性分析或反驳功能；没有则建议改写。
```
